# SOP_SYSTEM_OVERVIEW.md

Complete protocol for managing, duplicating, maintaining, and growing the 4_S0VRC3 symbolic operating system.

## CORE PHASES

### PHASE 0: GENESIS
- Run `initialize_vault.py` to build the symbolic folder structure.
- Use `generate_uid.py` to name everything legally.

### PHASE 1: ACTIVE USE
- Drop threads or ideas into `FL-TR` or `XP-PR`.
- Use `sort_router.py` to route, `tag_suggestor.py` to label.
- Use `rename_uid.py` after dumping files.

### PHASE 2: MAINTENANCE
- Use `uid_check.py` and `uid_indexer.py` weekly.
- Update local `UID_MAP.md` and `UIDS_MASTER_INDEX.md`.

### PHASE 3: ARCHIVING
- Run `vault_freeze.py` monthly or after a major build.
- Save output to `RC-VK_00-00_archives/fossils/`.

### PHASE 4: CLONING / FORKING
- Use `CLONE_PROTOCOL.md` to duplicate the system or fork for new use.
- Rename all folders/files using UID law.
- Begin again from `initialize_vault.py` if needed.

## SCRIPT INDEX
